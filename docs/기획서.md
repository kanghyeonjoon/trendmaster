# 유튜브 기획·대본 도우미 (가칭: TrendMaster Studio) 기획서

> 작성일: 2026-06-09
> 기존 앱(뉴스 트렌드 마스터 Pro)을 확장하여, 트렌드 수집부터 유튜브 기획안·대본 생성까지 한 흐름으로 잇는 도구.

---

## 1. 한 줄 요약

뉴스 트렌드를 모으고(이미 구현됨), 그 소재로 **유튜브 기획안과 대본을 Claude API로 자동 생성**해주는 프로그램. 쇼츠와 롱폼을 모두 지원한다.

### 확정된 방향 (2026-06-09)

- **타겟 시청자**: 병원 원장(개원의·병원 경영진). 소재·톤·예시를 병원 경영/마케팅/세무·노무/의료 트렌드 맥락에 맞춘다.
- **롱폼 기본 분량**: 10분 (한국어 내레이션 기준 약 2,800~3,200자)
- **사용 모델**: `claude-sonnet-4-6` (비용 절감 우선)
- **히스토리 저장**: 1차(MVP)에 포함

## 2. 만드는 이유 (문제 정의)

- 유튜브 영상의 성패는 **소재 선정**과 **대본의 후킹**에서 크게 갈린다.
- 트렌드를 찾는 일과 그걸 영상 기획으로 옮기는 일 사이에 단절이 있다 (기사 보고 → 직접 머리 싸매고 기획).
- 이 도구는 **트렌드 → 소재 → 기획안 → 대본**을 한 화면에서 끊김 없이 연결해, 기획에 드는 시간을 줄인다.

## 3. 핵심 사용자 흐름

```
[1] 트렌드 수집        (이미 있음) 뉴스 RSS + 워드클라우드
       ↓
[2] 소재 선정          기사 목록에서 영상화할 주제 선택
       ↓
[3] 영상 형태 선택      쇼츠(60초) 또는 롱폼(5~15분)
       ↓
[4] 기획안 생성        Claude API → 제목 후보, 후킹, 타겟, 구성 개요
       ↓
[5] 대본 생성          Claude API → 형태별 전체 멘트 대본
       ↓
[6] 내보내기           복사 / TXT 다운로드 / (선택) 저장
```

## 4. 기능 명세

### 4.1 MVP (1차 — 반드시 구현)

| # | 기능 | 설명 |
|---|------|------|
| F1 | 소재 선택 | 기존 뉴스 목록 행에 "이 소재로 기획" 버튼/선택 |
| F2 | 형태 선택 | 쇼츠 / 롱폼 라디오 버튼 |
| F3 | 기획안 생성 | Claude API 호출 → 제목 후보 3~5개, 한 줄 후킹, 타겟 시청자, 3단 구성 개요 |
| F4 | 대본 생성 | 기획안 기반 전체 대본 (형태별 분량/톤 자동 적용) |
| F5 | 내보내기 | 결과 복사 + TXT 다운로드 |
| F6 | API 키 입력 | 사이드바에서 Claude API 키 입력 (세션 보관) |
| F7 | 히스토리 저장 | 생성한 기획안·대본을 로컬 파일(CSV/JSON)에 저장하고 목록에서 다시 열기 |

### 4.2 2차 (확장)

- 채널 톤 설정: 정보전달형 / 친근한 말투 / 자극적 후킹형
- 썸네일 문구 + 영상 설명란 + 해시태그 자동 생성
- 직접 입력 소재 (뉴스 외에 내가 주제를 타이핑)
- 대본 재생성 / 부분 수정 ("인트로만 다시")

## 5. 형태별 대본 규격

### 쇼츠 (60초 이내)
- **분량**: 약 150~200자 (말하기 기준 45~55초)
- **구조**: 강한 후킹(3초 내) → 핵심 정보 1~2개 → 마무리 한 방
- **톤**: 빠르고 임팩트 있게, 군더더기 제거
- **추가물**: 자막용 짧은 문장, 화면 자막 포인트 표시

### 롱폼 (기본 10분, 5~15분 조절 가능)
- **분량**: 10분 기준 약 2,800~3,200자 (분량 선택 가능하게, 기본값 10분)
- **구조**: 인트로(후킹+예고) → 본론 2~4꼭지 → 아웃트로(요약+구독 유도)
- **톤**: 병원 원장 대상 설명형. 경영/마케팅 인사이트 위주, 흐름 있는 전개
- **추가물**: 챕터(타임스탬프) 구분, 각 꼭지 소제목

## 6. 기술 구성

- **프론트/실행**: Streamlit (기존 그대로)
- **트렌드 수집**: Google News RSS (기존 `get_news` 재사용)
- **AI 생성**: Anthropic Claude API (`anthropic` 파이썬 SDK)
  - 모델: `claude-sonnet-4-6` (비용 절감 우선)
  - 프롬프트: 형태(쇼츠/롱폼) + "병원 원장 타겟" 맥락 + 기사 내용을 조합한 시스템/유저 메시지
  - 비용 관리: 입력 기사 본문은 핵심만 추려 전달, 최대 토큰 제한 설정
- **추가 의존성**: `anthropic`
- **히스토리 저장소**: 로컬 파일 (JSON 또는 CSV), 별도 DB 없음
- **API 키 관리**: 사이드바 입력 또는 환경변수 `ANTHROPIC_API_KEY` (코드/저장소에 키 하드코딩 금지)

## 7. 화면 구성안 (Streamlit)

```
[사이드바]
  - Claude API 키 입력
  - 영상 형태: ◉ 쇼츠  ○ 롱폼
  - (2차) 채널 톤 선택
  - (2차) 롱폼 분량 슬라이더

[메인]
  탭1: 📰 트렌드 (기존) — 뉴스 목록 + 워드클라우드
        └ 각 기사에 "✍️ 이 소재로 기획" 버튼
  탭2: 🎬 기획·대본
        ├ 선택된 소재 표시
        ├ [기획안 생성] 버튼 → 결과 카드
        ├ [대본 생성] 버튼 → 대본 출력
        └ 복사 / TXT 다운로드
```

## 8. 개발 순서 (제안)

1. **0단계**: `anthropic` 의존성 추가, API 키 입력 UI
2. **1단계**: 기획·대본 탭 골격 + 소재 전달 (트렌드 → 기획 탭)
3. **2단계**: 기획안 생성 (F3) — 프롬프트 작성 + 출력 렌더링
4. **3단계**: 대본 생성 (F4) — 쇼츠/롱폼 분기
5. **4단계**: 내보내기 (F5)
6. **5단계**: 2차 기능 순차 추가
